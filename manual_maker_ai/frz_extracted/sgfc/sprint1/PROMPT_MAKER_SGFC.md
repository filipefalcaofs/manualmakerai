A property `Lista` e `Valores` DEVEM usar CR+LF real como separador.
NO BANCO: armazenar com CHAR(13)+CHAR(10) entre os itens.
```sql
-- CORRETO:
UPDATE FR_PROPRIEDADE SET PRO_VALOR = CONCAT('Direto', CHAR(13), CHAR(10), 'Indireto')

-- ERRADO (causa java.util.NoSuchElementException):
UPDATE FR_PROPRIEDADE SET PRO_VALOR = 'Direto\r\nIndireto'  -- literal, nao funciona
```

Lista e Valores DEVEM ter a MESMA quantidade de itens (correspondencia 1:1).

## 7. ERROS CONHECIDOS — NAO REPETIR

| Erro | Causa | Solucao |
|---|---|---|
| `NoSuchElementException` no HTMLComboBox | Lista/Valores com separador errado | Usar CHAR(13)+CHAR(10) real, nunca literal `\r\n` |
| Componentes empilhados | PosicaoX/PosicaoY iguais | Usar classes col-md-N para layout responsivo |
| Grid sem dados | Falta CodigoForm no grid | Criar CRUD filho e vincular via CodigoForm+Parametros |
| CheckBox nao salva | Property errada | Usar ValorCheck/ValorUnCheck (NAO ValorMarcado) |
| TextoRico nao funciona | Valor "True" | Usar valor "1" (numero) |
| Form nao abre | Navegacao=FALSE sem save | Manter Navegacao=TRUE e ocultar toolbar via CSS |
